## Usage

Commitlint checks if your commit messages meet the [conventional commit
format](https://www.conventionalcommits.org/). 

Example of valid commit messages:

```txt
feat: add new feature
fix: resolve bug in component
docs: update README
chore: update dependencies
style: format code
```

When configured with husky's [commit-msg
hook](https://commitlint.js.org/guides/local-setup.html), it will automatically
validate commit messages: 

```sh
# .husky/commit-msg

npx --no -- commitlint --edit $1
```
